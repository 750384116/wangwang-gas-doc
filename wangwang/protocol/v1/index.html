# 1. 完整协议格式

# 1.1. 下位机上传

ID-length ID Signal 模式 定位信息 EventType Type Command Data-Length Data 校验
1byte Nbyte 1byte 1byte Nbyte 1byte 2byte 1byte 2byte Nbyte 2byte

# 1.2. 上位机下发

ID-length ID Signal EventType Type Command Data-Length Data 校验
1byte Nbyte 1byte 1byte 2byte 1byte 2byte Nbyte 2byte

# 1.3. 字段描述

ID-length：D的长度

ID：DTU的唯一ID，默认使用模块的IMEI

​ signal: 信号值

​ 模式：0 GSM、1 CDMA、2 Nbiot

​ 定位信息：模式0 4byte （此次默认0）

​ byte[0]-byte[1] 小区号

​ byte[2]-byte[3] 基站号

​ EventType：00正常上报数据、01 开机信息、02 关机信息

​ Type：设备类型，默认写00

Command：请求类别（参考）

​ Data：数据块

​ Data-Length：数据块长度

校验：CRC16/modbus；例如数据为：01 02 03 04 校验= 2b a1

​

​ 规约内数据采用高字节在前 低字节在后(Big-Endian)

​ 校验计算Type~Data (包含Type和Data)

# 2. 数据包解析--设备上报

# 2.1. 开机、关机信息（Command：0x00）

QCCID 电池电压(DTU自带电池) 透传模块版本号
20byte 4byte(float) 1byte

Length：25byte

QCCID：预留信息

# 2.2. 上传报警数据（Command：0x01）

版本号 电池电压 控制器状态 回路总数 回路号 探测器最大总数 开始地址 探测器总数量 当前回路信息
1byte 2byte 2byte 1byte 1byte 1byte 1byte 1byte n*3byte

a) Length：10+n*3

b) 电池电压：除以100 例如 2678 ==>26.78V

c) 控制器状态：

Bit15: 1 总线短路 0 正常

Bit14-Bit8 保留

Bit7：1 主电工作中 0 备电工作中

Bit6：1 充电中 0 未充电

Bit5：1 主电故障 0 主电正常

Bit4：1 备电故障 0 备电正常

Bit3：1 充电故障 0 充电正常

Bit2：1 发生报警 0 未发生报警

Bit1：1 发生故障 0 未发生故障

Bit0: 保留

0xffff 代表控制器离线

d) 回路总数：取值1-4（默认1回路，出现多回路时需跟平台侧沟通设置情况）

e) 回路号：取值1-4（默认1回路，出现多回路时需跟平台侧沟通设置情况）

f) 探测器最大总数：按厂商配置（一般情况不超过64）

g) 开始地址：从1开始（探测器地址）

h) 探测器总数量：按厂商配置（一般情况不超过32）

i) 回路信息：每个探测器3bytes（探测器状态0xff 代表探测器为非法的）

  1. byte[0] 探测器状态
Bit7 1 离线 0 正常
Bit6-bit5 0 0 *1 01 *0.1 10 *0.01 11 *0.001
Bit4-bit3 00 %LEL 01 PPM 10 %VOL 11 其他
Bit2-bit0 001 正常 010 报警(低报) 011 报警(高报) 100 传感器离线 101 IIC离线 110 探测器版本错误 111 屏蔽！！！
  1. byte[1]-byte[2]：探测器浓度

bit15 ：1寿命到期、0正常；bit14-bit0：浓度

示例: 0F383631303530303430343230363538130000000000000005010010000000008001010201020100000100004D1B

# 2.3. 平台下发获取命令，设备响应结果（Command：0x02）

Byte[0]-byte[3] taskId

Byte[4]-byte[7] 正常状态下上传周期 (秒)

Byte[8]-byte[11] 报警后上传周期 (秒)

Byte[12] 探测器数量N 返回探测器的数量

Byte[13] 探测器地址号1

Byte[14]-byte[15] 探测器报警点1，0代表无效，高位在前低位在后，10进制整数的16进制表现形式

Byte[16] 探测器突变阈值1，10进制整数的16进制表现形式

Byte[17]….. 探测器地址号2

…

Byte[..]….. 探测器地址号N

# 2.4. 平台下发设置命令，设备响应结果 （Command：0x03)

Byte[0]-byte[3] taskId

Byte[4]-byte[7] 正常状态下上传周期 (秒)

Byte[8]-byte[11] 报警后上传周期 (秒)

Byte[12] 探测器数量N

Byte[13] 探测器地址号1

Byte[14]-byte[15] 探测器报警点1 0代表无效

Byte[16] 探测器突变阈值1

Byte[17]….. 探测器地址号2

…

Byte[..]….. 探测器地址号N

# 2.5. 平台下发复位命令，设备响应结果 （Command：0x06）

数据同功能码3，预留功能

# 2.6. 平台下发消音命令，设备响应结果（Command：0x07）

数据同功能码3，预留功能

# 2.7. 平台下发命令参数错误，设备响应结果 （Command：0x7e）

Length:5byte

Byte[0]-byte[3] taskID

Byte[4] 下发的功能码

# 2.8. 平台下发命令，设备不支持 （Command：0x7f）

Length:5byte

Byte[0]-byte[3] taskID

Byte[4] 下发的功能码

# 3. 数据包解析--平台下发

# 3.1. 获取设备参数 （Command：0x82）

Length:4

Byte[0]-byte[3] taskId

示例 : 0f3335393336393038313133323736390000000082000400000008fe58

# 3.2. 设置参数 （Command：0x83）

Byte[0]-byte[3] taskId

Byte[4]-byte[7] 正常状态下上传周期 (秒)

Byte[8]-byte[11] 报警后上传周期 (秒)

Byte[12] 探测器数量N (这里是需要修改的探测器的数量)

Byte[13] 探测器地址号1

Byte[14]-byte[15] 探测器报警点1

Byte[16] 探测器突变阈值1

Byte[17]….. 探测器地址号2

…

Byte[..]….. 探测器地址号N

# 3.3. 复位（Command：0x86）

Length:4byte

Byte[0]-byte[3] taskID

# 3.4. 消音（Command：0x87）

Length:4byte

Byte[0]-byte[3] taskID
