    # 1. 适用范围

    # 1.1. 系统式探测器

    # 1.2. 工业及商业用途点型可燃气体探测器

    • 控制器满足GB16808-2008
    • 探测器满足GB15322.1-2019

    # 1.3. 家用可燃气体探测器

    • 满足GB15322.4-2019

    # 2. 支持的通讯方式

    • 电信Nbiot标准版
    • 电信Nbiot基础版(Aep)
    • Http API
    • TCP

    # 3. 设备功能的约定

    # 3.1. 工业及商业用途、家用报警器通用功能

    • 数据传输中需要对如下工作参数进行设置
    参数名称 单位 缺省值 设置范围 使用方式
    平安报上传频率 分钟 120 1-65535 按照此时间间隔向后台上报数据
    浓度突变阈值 %LEL 5 0.1-25.5 该数值作为判断“浓度变化”的阈值条件。
    警戒后上传频率 分钟 10 1-65535 超警戒值后，按照此时间间隔向后台上报数据
    一级警戒浓度（低报） %LEL 15 超过警戒值，立即上报数据
    二级警戒浓度 %LEL 25 超过警戒值，立即上报数据
    三级警戒浓度（高报） %LEL 50 超过警戒值，立即上报数据

    • 通讯模块正常会按照“平安报上传频率”上报数据；

    • 当浓度变化超过约定的“浓度突变阈值”时，将会立即上报数据；

    • 超警戒浓度时，立即上报数据，并按照“警戒后上传频度”上报数据；

    • 模块需关闭PSM模式（家用除外）；

    • 如设备只有报警点无具体浓度值时，需上报探测器报警状态；

    • 平安报上传频率、警戒后上传频率保存在控制器上，设备下多路探测器共享；

    • 一二三级警戒浓度值、浓度突变阈值为探测器私有属性，每台探测器可以单独设置；

    • 正常数据上报时，设备按照平安报频率将所有探测器数据包含在一个数据包中发送给平台；

    • 浓度超阈值变化上报，一段时间内采集到的2个相邻时间点的浓度值相减取绝对值，若大于浓度变化阈值，则立马将所有探测器数据包含在一个数据包中发送给平台；

    • 浓度超警戒值上报，当设备下某台探测器浓度达到了报警点， 则按照警戒后的上报频率， 将设备所有探测器数据包含在一个数据包中发送给平台；

    • 获取设备参数,服务器下行命令向设备获取参数设置，设备接收到指令后向平台上报参数数据；

    • 设置设备参数,服务器下行命令修改设备运行参数，设备接收到指令后进行修改操作，然后向平台上报修改完毕后的参数数据（注：修改第n地址号探测器，不需要带上所有探测器的参数配置，只需要地址号n+该探测器的参数值即可，例如修改地址号2的警戒点和阈值，下行命令为<……地址号2+地址2探测器报警点+地址号2探测器阈值……>）；

    • 设备主电异常、启用备电时将设备所有探测器数据包含在一个数据包中发送给平台；

    • 设备开机、关机时将设备基础数据包含在一个数据包中发送给平台；

    # 3.2. 工业及商业用途点型可燃气体控制器及探测器

    • 探测器的报警动作值：测量范围在3~100%LEL的探测器，报警动作值与报警设定值之差的绝对值不应大于3%LEL；
    • 可燃气体报警声信号应能手动消除，当再次有可燃气体报警信号输入时，应能再次启动；
    • 应能显示当前可燃气体报警部位的总数；
    • 应能区分最先报警部位；
    • 后续报警部位应按照报警时间顺序连续显示，当显示区域不足以显示全部报警部位时，应按顺序循环显示，同时应设置手动查询按钮；
    • 具有报警历史事件记录功能，至少记录999条相关信息（报警、通讯、基站交互等数据），且在控制器断电后能保持信息14d；
    • 探测器内部应具有读取接口，使用可燃气体报警控制器或探测器报警历史信息记录读取装置应能对探测器的报警历史记录完整读取；
    • 通过控制器可改变与其连接的可燃气体探测器报警设定值时，改报警设定值应能在控制器上手动可查；
    • 自检功能：控制器应能检查本机的可燃气体报警功能（以下称自检），控制器在执行自检功能期间，受其控制的外接设备和输出接点均不应动作。控制器自检时间超过1min或其不能自动停止自检功能时，控制器的自检功能应不影响非自检部位和控制器本身的可燃气体报警功能。
    • 电源功能：控制器的电源部分应具有主电源和备用电源转换装置，当主电源断电时，能自动转换到备用电源，主电恢复时，能自动转换到主电源，有主备电工作状态指示；
    • 当主电源断电，备用电源不能保证控制器正常工作时，控制器应发出故障声信号并能保持1h以上；

    # 3.3. 家用可燃气体探测器

    • 探测器内部应具有报警历史记录功能，历史记录在探测器掉电后应能保存。历史记录的类型和条数应满足以下要求：
      • 探测器报警记录：不少于200条；
      • 探测器报警恢复记录：不少于200条；
      • 探测器故障记录：不少于100条；
      • 探测器故障恢复记录：不少于100条；
      • 探测器掉电记录：不少于50条；
      • 探测器上电记录：不少于50条；
      • 气体传感器失效记录：不少于1条
    • 探测器内部应具有读取接口，使用可燃气体报警控制器或探测器报警历史信息记录读取装置应能对探测器的报警历史记录完整读取；

    # 3.4. 多点位探测器(探测器数量大于32)数据上报

    • 当报警器接入探测器数量大于32时,数据需要分包上报
    • 每一包上报数据最多包含32个连续的探测器数据(例如64探头报警器,1~32号探头第一包上报,33~64探头第二包上报,以此类推)
    • 每一包数据必须符合完整的协议格式
